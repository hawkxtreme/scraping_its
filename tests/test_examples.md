# Примеры команд для тестирования улучшений

Этот файл содержит примеры команд для тестирования каждого улучшения, внесенного в проект.

## Базовый URL для тестирования
```
https://its.1c.ru/db/cabinetdoc
```

## 1. Улучшенное логирование и информативность ошибок

### Тестирование расширенного логирования
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --debug
```
Ожидаемый результат:
- Создается лог-файл в `out/cabinetdoc/script_log.txt`
- В консоли отображается детальная отладочная информация
- Логируются все шаги процесса с временными метками

### Проверка лог-файла
```bash
cat out/cabinetdoc/script_log.txt
```
Ожидаемый результат:
- Видны записи уровня DEBUG, INFO, WARNING, ERROR
- Записи содержат временные метки и детальную информацию

## 2. Поддержка формата DOCX

### Тестирование сохранения в формате DOCX
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format docx --max-articles 2
```
Ожидаемый результат:
- Создаются DOCX файлы в `out/cabinetdoc/docx/`
- Файлы содержат форматированный текст с заголовками, таблицами и списками

### Проверка DOCX файлов
```bash
ls -la out/cabinetdoc/docx/
```
Ожидаемый результат:
- Видны созданные DOCX файлы

## 3. Оптимизация параллельной индексации

### Тестирование параллельной обработки
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 5 --parallel 3
```
Ожидаемый результат:
- Статьи обрабатываются в 3 параллельных потока
- В консоли отображается прогресс-бар
- В логах указывается статистика работы воркеров

### Сравнение с последовательной обработкой
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 5 --parallel 1
```
Ожидаемый результат:
- Статьи обрабатываются последовательно
- Время выполнения должно быть больше, чем при параллельной обработке

## 4. Система кэширования

### Первый запуск с кэшированием
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --use-cache
```
Ожидаемый результат:
- Статьи сохраняются и кэшируются
- Создается директория `.cache` в `out/cabinetdoc/`

### Проверка статистики кэша
```bash
python main.py --cache-stats
```
Ожидаемый результат:
- Отображается статистика кэша: размер, количество записей, возраст

### Второй запуск (должно использовать кэш)
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --use-cache
```
Ожидаемый результат:
- Статьи извлекаются из кэша (быстрее первого запуска)
- В логах указывается об использовании кэша

### Очистка кэша
```bash
python main.py https://its.1c.ru/db/cabinetdoc --clear-cache
```
Ожидаемый результат:
- Кэш очищается
- В логах указывается об очистке кэша

## 5. Консольные аргументы для настройки таймаутов и ограничений

### Тестирование настройки таймаутов
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 2 --page-timeout 120000 --login-timeout 90000
```
Ожидаемый результат:
- Используются увеличенные таймауты для загрузки страниц и входа
- В отладочном режиме отображаются настройки таймаутов

### Тестирование настройки ограничений скорости
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 2 --request-delay 1.0 --max-retries 5
```
Ожидаемый результат:
- Используется задержка 1 секунда между запросами
- Увеличено количество повторных попыток до 5
- Общее время выполнения увеличивается из-за задержек

## 6. Режим обновления существующих файлов

### Создание индекса без скрапинга
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --no-scrape
```
Ожидаемый результат:
- Создается индекс статей без скрапинга контента
- В логах указывается о выходе без скрапинга

### Тестирование режима обновления
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --update
```
Ожидаемый результат:
- Проверяются хэши существующих статей
- Обновляются только измененные статьи
- В логах указывается количество новых, измененных и неизмененных статей

### Принудительное обновление всех статей
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 3 --update --force-reindex
```
Ожидаемый результат:
- Переиндексируются все статьи, независимо от изменений
- В логах указывается о принудительном переиндексировании

## 7. Расширение возможностей фильтрации и выбора статей

### Тестирование фильтрации по заголовку
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --filter-by-title "Отчет" --max-articles 2
```
Ожидаемый результат:
- Выбираются только статьи, в заголовке которых есть слово "Отчет"
- В логах указывается количество отфильтрованных статей

### Тестирование исключения по заголовку
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --exclude-by-title "Архив" --max-articles 2
```
Ожидаемый результат:
- Исключаются статьи, в заголовке которых есть слово "Архив"
- В логах указывается количество отфильтрованных статей

### Тестирование ограничения количества статей
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --max-articles 5
```
Ожидаемый результат:
- Обрабатываются только первые 5 статей
- В логах указывается об ограничении количества

### Тестирование пропуска первых статей
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --skip-first 3 --max-articles 2
```
Ожидаемый результат:
- Пропускаются первые 3 статьи
- Обрабатываются следующие 2 статьи
- В логах указывается о пропуске статей

### Тестирование случайной выборки
```bash
python main.py https://its.1c.ru/db/cabinetdoc --format json --random-sample 3
```
Ожидаемый результат:
- Случайно выбираются 3 статьи из всех доступных
- В логах указывается о случайной выборке

## Примечания

1. Для выполнения тестов требуется, чтобы сервис browserless был запущен и доступен
2. Учетные данные для входа на сайт должны быть указаны в файле `.env`
3. Некоторые тесты могут занимать несколько минут в зависимости от скорости соединения
4. При первом запуске может потребоваться установка браузера для Playwright:
   ```bash
   playwright install chromium
